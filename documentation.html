ShreeNath ElectroManager

A comprehensive role-based electronic retail management system with inventory control, supplier management, and customer ordering.

📋

Synopsis

Project Overview — ≤ 100 words

ElectroManager is a web-based electronic retail management system designed for ShreeNath enterprises. It provides a centralised platform where Admins manage inventory, suppliers, users and orders; Suppliers list and update their product catalogue; and Customers browse products and place secure orders. Built with HTML, CSS, JavaScript and backed by Supabase for cloud authentication and LocalStorage for client-side persistence, it delivers a seamless, role-gated experience through a modern glassmorphism UI.

Modules & Description

Core system components — each ≤ 100 words

🔐

Authentication Module

Handles customer registration, login, and admin access via a flip-card UI. Supports Supabase cloud auth with LocalStorage fallback. Role assignment (Admin / Supplier / Customer) is done at registration.

All Roles
📊

Admin Dashboard

Displays key KPIs — total products, verified users, and net revenue — with a real-time revenue trend chart (Chart.js) and an inventory snapshot of top products.

Admin
📦

Inventory Management

Allows admins to add, edit, and delete products with photo URL, price, stock count, and description. Shows stock-level badges (red = low, green = sufficient) for quick status checks.

Admin
🚚

Supplier Directory

Manages vendor profiles including company name, contact person, email, phone, and product categories. Admins can create, update, or remove supplier records from a centralised directory.

Admin
👥

User Management

Enables admins to view all registered users, modify their roles (Admin / Supplier / Customer), and delete accounts. New users can also be manually created from this module.

Admin
🧾

Orders Module

Admin sees a master list of all platform transactions. Customers view their personal purchase history. Orders record amount, payment method (UPI, Card, COD, Net Banking), and date.

Admin + Customer
🏪

Supplier Brand Portal

Suppliers can add and manage their own brand products independently. They also maintain a self-editable profile with company details, visible only to the Admin and themselves.

Supplier
🛒

Customer Market

Customers browse the product gallery, select items, and checkout using multiple payment options. A purchase history section tracks all past orders securely for the logged-in customer.

Customer

Hardware & Software Requirements

Minimum system configuration — ≤ 100 words each

🖥️ Hardware Requirements

  • Processor: Intel Core i3 / AMD Ryzen 3 or higher
  • RAM: Minimum 4 GB (8 GB recommended)
  • Storage: At least 500 MB free disk space
  • Display: 1280 × 720 resolution or above
  • Network: Active internet connection for Supabase cloud auth
  • Input: Standard keyboard and mouse / touchscreen

💻 Software Requirements

  • OS: Windows 10/11, macOS 12+, or Ubuntu 20+
  • Browser: Google Chrome 110+, Firefox 111+, or Edge 110+
  • Runtime: Node.js v18+ (for local dev server)
  • Backend: Supabase (cloud) — no local DB server needed
  • Libraries: Chart.js 4, Supabase JS v2, Ionicons 5
  • Tools: VS Code (recommended), Git for version control
